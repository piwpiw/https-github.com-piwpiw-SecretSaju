# Result Output Design Spec (2026-03-07)

목적:
- 입력 후 결과가 생성되는 각 페이지의 출력 구조를 상세 설계로 고정한다.
- 화면 카피, 프리미엄 확장, 저장/공유, QA 기준을 한 문서에서 관리한다.

적용 범위:
- `/saju`
- `/fortune`
- `/compatibility`
- `/relationship/[id]`
- `/relationship/[id]/vs`
- `/daily`
- `/calendar`
- `/tojeong`
- `/tarot`
- `/dreams`
- `/psychology/module/[id]`
- `/naming`
- `/gift`
- `/inquiry`

공통 규칙:
- 결과는 `요약 -> 근거 -> 실행 조언 -> 프리미엄 확장` 순서로 노출
- 첫 화면에서 3초 안에 핵심 결과 1개가 보여야 함
- 프리미엄 잠금은 “무엇이 더 열리는지”를 구체적으로 설명해야 함
- 저장 가능한 결과는 모두 `제목`, `한 줄 요약`, `핵심 지표`, `상세 payload`를 가져야 함
- 이모지는 의미를 강화하는 용도로만 사용하며, 결과 의미와 직접 연결되어야 함

## 1. `/saju`

### 입력
- 생년월일
- 출생시간
- 성별
- 양력/음력
- 윤달 여부

### 계산 계층
- 사주 원국 계산
- 오행 분석
- 십성 분석
- 신강/신약 분석
- 격국 후보
- 용신 후보
- 운세 상호작용
- AI 확장 서사

### 기본 결과 블록
1. 헤더 요약
- `📜 공식 사주 분석 리포트`
- 이름/나이대/시간 입력 상태
- 핵심 한 줄 해석

2. 구조 블록
- 년주/월주/일주/시주
- 오행 분포
- 십성 분포
- 십이운성

3. 해석 블록
- `🧭 만세력 기반 기본 풀이`
- `✨ 시크릿 핵심 해석`
- `🔥 신강·신약 밸런스`
- `🧩 격국 후보`
- `🌿 용신/희신/기신`

4. 실행 블록
- 연애/재물/직업/기본 관점 전환
- 현재 세운/월운/일운 상호작용
- 행동 가이드 2~3개

### 프리미엄 확장
- 더 자세한 용신 전략
- 십성 직업 코드
- 관계 패턴
- 돈 흐름
- 실행 타이밍
- 멤버십 티저

### 저장 필드
- title: `{이름} 사주 리포트`
- summary: 핵심 한 줄 해석
- oneLineSummary: 공유용 짧은 문장
- categories:
  - 오행
  - 십성
  - 강약
  - 관계
  - 재물
  - 직업

### QA 기준
- 시간 미입력 시 보정 문구 노출
- 오행/십성/강약 블록 중 최소 1개 이상이 빈 상태 없이 렌더
- 프리미엄 CTA가 첫 결과 화면 안에서 보임

## 2. `/fortune`

### 입력
- 연도
- 기본 프로필 또는 기준 정보

### 기본 결과 블록
1. `🗓️ 올해의 총운`
- 한 줄 총평
- 상승/유지/주의 톤

2. 영역별 카드
- 연애
- 재물
- 커리어
- 건강/생활

3. 실전 조언
- 지금 해야 할 것
- 피해야 할 과속

### 프리미엄 확장
- 월별 디테일
- 리스크 구간
- 기회 구간

### 저장 필드
- year
- summary
- categories[] with `label`, `score`, `tone`, `reason`, `action`

### QA 기준
- 총평 1개, 카테고리 3개 이상, 액션 팁 1개 이상

## 3. `/compatibility`

### 입력
- 프로필 A
- 프로필 B
- 관계 유형

### 기본 결과 블록
1. `💞 궁합 점수`
- 총점
- 등급 메시지

2. 세부 축
- 감정 호흡
- 생활 리듬
- 갈등 포인트
- 장기 안정성

3. 조언
- 잘 맞는 방식
- 조심해야 할 패턴

### 프리미엄 확장
- 관계 심화 리포트
- 장기 전망
- 실전 대화 가이드

### 저장 필드
- score
- chemistry
- tension
- advice
- futurePredict

### QA 기준
- 총점과 설명 문구가 동시에 보여야 함
- 결과 후 공유 또는 상세 CTA가 보여야 함

## 4. `/relationship/[id]`, `/relationship/[id]/vs`

### 입력
- 저장 프로필
- 비교 대상

### 결과 블록
- `🫶 관계 요약`
- 상대와의 강점
- 긴장 포인트
- 행동 제안
- 미래 흐름

### 프리미엄 확장
- 관계 단계별 해석
- 감정 패턴 분석

### QA 기준
- 비교 결과와 다음 행동 CTA가 같이 보여야 함

## 5. `/daily`

### 입력
- 사용자 프로필 또는 기본 기준 정보

### 결과 블록
- `☀️ 오늘의 운세`
- 좋은 흐름
- 주의 포인트
- lucky signal
- 오늘의 행동 팁

### 프리미엄 확장
- 시간대별 포인트
- 관계/돈/일 미세 해석

### QA 기준
- 날짜, 요약, 행동 팁이 첫 화면에 보여야 함

## 6. `/calendar`

### 입력
- 월
- 날짜 선택

### 결과 블록
- 월간 캘린더
- 선택 날짜 상세
- `📅 오늘/선택일 포인트`
- 추천 행동

### 프리미엄 확장
- 일정 추천
- 알림/리마인더

### QA 기준
- 월간 그리드와 날짜 상세 패널이 모두 노출

## 7. `/tojeong`

### 입력
- 생년월일

### 결과 블록
- `🧭 월별 흐름`
- 각 월 점수
- 이유
- 주의점
- 실행 조언

### 프리미엄 확장
- 더 촘촘한 월간 리듬
- 인간관계/재물 관점 분기

### QA 기준
- 12개월 또는 요약 월간 흐름이 연속적으로 렌더

## 8. `/tarot`

### 입력
- 질문
- 모드
- 카드 선택/드로우

### 결과 블록
- `🃏 카드 해석`
- 현재 상황
- 감정 포인트
- 행동 조언

### 프리미엄 확장
- 카드 간 연결 해석
- 장기 방향

### QA 기준
- 카드명, 해석, 다음 행동 1개 이상

## 9. `/dreams`

### 입력
- 꿈 서술
- 감정/상황

### 결과 블록
- `🌙 꿈 해석`
- 상징 의미
- 감정 해석
- 현실 적용 포인트

### 프리미엄 확장
- 반복 패턴
- 무의식 테마

### QA 기준
- 상징/감정/행동 포인트 3요소가 함께 보여야 함

## 10. `/psychology/module/[id]`

### 입력
- 문항 응답

### 결과 블록
- `🧠 심리 요약`
- 성향 패턴
- 방어 패턴
- 개선 방향

### 프리미엄 확장
- deep report
- 행동 처방

### QA 기준
- 점수 또는 상태 + 문장형 해석 + 다음 단계 CTA

## 11. `/naming`

### 입력
- 이름 관련 조건

### 결과 블록
- `✍️ 이름 방향`
- 의미 요약
- 추천 포인트

### 프리미엄 확장
- 후보 비교
- 상세 이유

### QA 기준
- 후보 또는 방향성, 설명, CTA 동시 노출

## 12. `/gift`

### 입력
- 수신자
- 메시지
- 상품/금액

### 결과 블록
- `🎁 발송 결과`
- 주문 상태
- 링크/공유 경로

### QA 기준
- 결제 후 링크 또는 완료 상태가 명확해야 함

## 13. `/inquiry`

### 입력
- 카테고리
- 문의 내용

### 결과 블록
- `📩 접수 완료`
- 응답 기대 시점
- 후속 경로

### QA 기준
- 접수 확인 문구와 다음 행동이 동시에 보여야 함

## Cross-Page Copy Rule

- 첫 문장은 요약형
- 두 번째 문장은 원인형
- 세 번째 문장은 행동형
- 프리미엄 문장은 확장형

예시 구조:
- `🧭 지금 가장 강하게 드러나는 기운은 ...입니다.`
- `📚 그 이유는 오행/십성/관계 축에서 ...이 반복되기 때문입니다.`
- `✨ 따라서 지금은 ... 방식으로 움직이는 것이 유리합니다.`
- `🔐 프리미엄 해제 시 ...까지 더 자세히 확인할 수 있습니다.`
