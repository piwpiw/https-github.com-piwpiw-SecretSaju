### 🚧 Q2 2026: Viral Growth (Apr-Jun)

**Theme**: **Share & Grow**

**Goal**: 10K → 50K MAU through social features

#### Features

##### 1. **Share-to-Unlock Mechanics** 🔥
**Problem**: Users want full analysis but don't want to pay  
**Solution**: Share on Kakao/Insta → unlock detailed report

**User Flow**:
```
User gets basic saju (free)
  ↓
"Share to unlock full analysis"
  ↓
User shares on KakaoTalk
  ↓
Friends click link → sign up
  ↓
Original user gets full report unlocked
```

**Impact**: Viral coefficient 0.5 → 1.5 (projected)

---

##### 2. **Celebrity Matching** 🌟
**Problem**: People love comparing themselves to famous people  
**Solution**: "Find your saju twin" - match with celebrities

**Features**:
- Database of 100+ Korean/Global celebrities
- Match score (0-100%)
- Shareable result cards ("I'm 85% like IU!")

**Viral Potential**: High (meme-worthy)

---

##### 3. **Referral System** 🎁
**Problem**: CAC is high ($5+)  
**Solution**: Give jellies for referring friends

**Mechanics**:
- Refer a friend → both get 1 free jelly
- Friend makes first purchase → you get 2 more jellies
- Leaderboard for top referrers

**Impact**: CAC $5 → $3 (projected)

---

##### 4. **Daily Fortune Push** 📱
**Problem**: Low return rate after first visit  
**Solution**: Daily personalized fortune notifications

**Features**:
- Smart notification timing (8am local time)
- Personalized content (based on saju + current luck period)
- Rich notification (image + text)

**Impact**: DAU/MAU 10% → 25%

---

##### 5. **Kakao Story Templates** 📸
**Problem**: ShareThis link is boring  
**Solution**: Beautiful story templates for sharing

**Features**:
- 5+ templates (mystic, minimal, meme style)
- Auto-populate user data
- One-tap share to Kakao/Insta Story

**Impact**: Share rate 5% → 20%

---

**Q2 Success Metrics**:
- MAU: 50,000
- Viral Coefficient: > 1.2
- Share Rate: > 15%
- CAC: < $3

### 🔮 Q3 2026: AI & Personalization (Jul-Sep)

**Theme**: **Smart & Premium**

**Goal**: 100K MAU + increase ARPU ($3 → $5)

#### Features

##### 1. **AI Fortune Telling (GPT-4)** 🤖
**Problem**: Static descriptions feel impersonal  
**Solution**: AI-generated personalized fortunes

**Features**:
- Daily AI-written horoscope (based on saju + current events)
- Career advice (based on elements + job data)
- Relationship insights (compatibility + communication tips)

**Tech**: OpenAI GPT-4 API

**Impact**: ARPU $3 → $4 (premium feature)

---

##### 2. **Smart Recommendations** 🎯
**Problem**: Users don't know how to apply saju to real life  
**Solution**: Personalized recommendations

**Categories**:
- **Food**: Balance your elements (目 → eat green veggies)
- **Colors**: Lucky colors for today
- **Activities**: Auspicious timing (start new project on 甲子 day)
- **Products**: Partner with wellness brands

**Monetization**: Affiliate links (10-15% commission)

---

##### 3. **Luck Calendar** 📅
**Problem**: Users forget to check daily fortune  
**Solution**: Visual calendar with daily/monthly luck indicators

**Features**:
- Month view with color-coded days (good/neutral/bad)
- Tap day → detailed fortune
- Export to Google Calendar
- Smart reminders ("Tomorrow is lucky for decisions!")

**Impact**: Retention D30: 40% → 55%

---

##### 4. **Premium Interpretation** 💎
**Problem**: Basic users not converting  
**Solution**: Tiered content (basic → advanced → master)

**Tiers**:
- **Free**: Basic saju + element analysis
- **Smart (2,900원)**: + Luck periods + compatibility
- **Pro (9,900원)**: + AI insights + unlimited profiles
- **Master (29,900원)**: + 1-on-1 expert consultation

**Impact**: Conversion 10% → 15%

---

**Q3 Success Metrics**:

## Execution Backlog Upgrade (UI + Feature + Must-Show Elements)

Execution-ready screen contracts are centralized in [Execution Backlog](./execution-backlog-ko.md). This keeps `roadmap.md` focused on schedule and strategic flow while route-level must-show requirements stay in one file.

### Why this separation
- Route contract is now the single source of truth for screen-level acceptance.
- PM, engineering, and QA read the same contract before each sprint.
- Duplicate tables in multiple docs are removed to reduce drift.

### Usage
1. Before each sprint, open `execution-backlog-ko.md` and choose active rows.
2. Link each selected route to at least one test case in `USER_VERIFICATION.md`.
3. Update `execution-backlog-ko.md` when a route contract changes, then keep the roadmap update entry minimal.
