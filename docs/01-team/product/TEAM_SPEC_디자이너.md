## Rules

- Follow the established visual language before introducing new patterns.
- Keep interaction changes paired with implementation-ready guidance.
- Do not move core placement rules into design docs; link to the canonical structure doc instead.
- When a visual change affects behavior, align with engineering and QA.

## Deliverables

- Component and screen direction
- Visual QA notes
- Interaction polish guidance
- Design-to-implementation handoff notes
