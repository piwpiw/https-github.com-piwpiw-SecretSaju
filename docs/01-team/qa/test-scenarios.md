# QA Test Scenarios - Secret Saju

**Quality Assurance Testing Guide**

---

## 🎯 QA 원칙

1. **사용자 관점으로 테스트**: 실제 사용 시나리오 기반
2. **엣지 케이스 포함**: 극단적 입력값도 테스트
3. **환경 다양화**: 여러 브라우저/디바이스에서 검증
4. **회귀 테스트**: 기존 기능이 깨지지 않았는지 확인

---

## 📋 테스트 체크리스트

### 릴리스 전 필수 테스트 (Smoke Test)

**소요 시간**: 30분

- [ ] **홈페이지 로딩** - 3초 이내
- [ ] **사주 계산** - 정상 작동
- [ ] **카카오 로그인** - 성공
- [ ] **결제 플로우** - End-to-end 정상
- [ ] **모바일 반응형** - UI 깨지지 않음
- [ ] **에러 없음** - 콘솔에 critical error 없음

## PASS 기준 수치 정리

- 인증(3-1): OAuth 콜백 성공 시 `/dashboard` 이동 2.5초 이내(목표 2500ms), 실패 시 `/` 이동 2.4초 이내(목표 2400ms)
- OAuth 유효성 검사: `state` 길이 16~128, 정규식 일치, `code` 길이 8~256 정합성 위반 시 즉시 에러 라우팅
- MCP PKCE 무결성: `code_verifier` 길이/문자셋 미일치 시 바로 감사 로그(`invalid PKCE code_verifier`) 출력
- 콜백 중복 처리: 동일 state 재호출 시 단일 성공/실패만 허용
- UI 응답성: 에러 상세 문구 전체 보기 토글 시 텍스트가 끊김 없이 렌더링 유지
- 다중 로그인 버튼 상태: 한 provider 동작 중 다른 provider 버튼은 disabled 및 지역화된 안내문 출력
## QA Smoke 우선순위(결제/로그인)

1. 결제 실패 케이스 (missing params, amount mismatch, wallet update fail)
2. 결제 성공 idempotency (중복 콜백 재시도)
3. 로그인 콜백 에러 provider_error 전달
4. Kakao/MCP 세션 쿠키 정합성
5. 결제 알림/영수증 메일 발송 실패
