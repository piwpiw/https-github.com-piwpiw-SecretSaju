# QA Test Scenarios - Secret Saju

**Quality Assurance Testing Guide**

---

## 🎯 QA 원칙

1. **사용자 관점으로 테스트**: 실제 사용 시나리오 기반
2. **엣지 케이스 포함**: 극단적 입력값도 테스트
3. **환경 다양화**: 여러 브라우저/디바이스에서 검증
4. **회귀 테스트**: 기존 기능이 깨지지 않았는지 확인

---

## 📋 테스트 체크리스트

### 릴리스 전 필수 테스트 (Smoke Test)

**소요 시간**: 30분

- [ ] **홈페이지 로딩** - 3초 이내
- [ ] **사주 계산** - 정상 작동
- [ ] **카카오 로그인** - 성공
- [ ] **결제 플로우** - End-to-end 정상
- [ ] **모바일 반응형** - UI 깨지지 않음
- [ ] **에러 없음** - 콘솔에 critical error 없음

---

## 🧪 기능별 테스트 시나리오

### 1. 사주 계산 엔진

#### 시나리오 1-1: 양력 생일 입력 (정상 케이스)

**Given**: 사용자가 메인 페이지에 있음  
**When**: 
1. 생년월일: `1990-01-15`
2. 출생 시간: `14:30`
3. 성별: `남성`
4. 달력 유형: `양력`
5. "사주 보기" 클릭

**Then**:
- [x] 사주팔자 4기둥이 정확히 표시됨
- [x] 오행 분석 (木火土金水) 정상
- [x] 로딩 시간 < 3초
- [x] 에러 없음

**Expected Output**:
```
연주(年柱): 己巳 (기사)
월주(月柱): 丁丑 (정축)
일주(日柱): [계산값]
시주(時柱): [계산값]
```

---

#### 시나리오 1-2: 음력 생일 입력

**Given**: 사용자가 입력 폼에 있음  
**When**:
1. 생년월일: `1990-01-15`
2. 달력 유형: `음력` 선택
3. 출생 시간: `12:00`
4. 성별: `여성`
5. 제출

**Then**:
- [x] 음력 → 양력 변환 정확
- [x] 변환된 양력 날짜 표시됨
- [x] 사주 계산 정상

---

#### 시나리오 1-3: 시간 모름

**When**:
1. 생년월일: `1995-05-20`
2. 출생 시간: `모름` 체크박스 선택
3. 제출

**Then**:
- [x] 시주(時柱) 없이 계산됨
- [x] "출생 시간을 모르면 시주는 계산되지 않습니다" 안내 표시
- [x] 나머지 3기둥은 정확

---

#### 시나리오 1-4: 엣지 케이스 - 미래 날짜

**When**:

2. 제출

**Then**:

- [x] 폼이 리셋되지 않음 (사용자 입력 보존)

---

#### 시나리오 1-5: 엣지 케이스 - 1900년 이전

**When**:
1. 생년월일: `1899-12-31`
2. 제출

---

#### 시나리오 1-6: 섬머타임 기간 (1948-1988)

**When**:
1. 생년월일: `1960-07-15` (한국 섬머타임 기간)

3. 제출

### 2. 프로필 관리

#### 시나리오 2-1: 프로필 저장

**Given**: 사용자가 로그인 상태

2. "프로필 저장" 클릭
3. 이름: `나`
4. 관계: `본인` 선택
5. 저장

**Then**:
- [x] "저장되었습니다" 토스트 알림
- [x] 마이페이지 → 프로필 목록에 표시됨

#### 시나리오 2-2: 프로필 불러오기

**Then**:
- [x] 저장된 사주 결과 표시
- [x] 재계산 없이 캐시된 데이터 사용

### 4. 결제 시스템

3. Toss 결제 위젯에서 결제 완료

**Then**:
- [x] 결제 성공 페이지로 리다이렉트

- [x] `jelly_transactions` 테이블에 기록됨

#### 시나리오 4-2: 결제 실패

#### 시나리오 4-4: 젤리 부족

## 🤖 자동화 테스트

### Unit Tests (Vitest)

**위치**: `*.test.ts` 파일

**실행**:
```bash
npm run test
```

**커버리지**:
```bash
npm run test:coverage
```

**목표**: 70% 이상

## 🐛 버그 리포팅

### 버그 템플릿

```markdown
## Bug Report

**Title**: [간단한 설명]

**Steps to Reproduce**:
1. [Step 1]
2. [Step 2]
3. [Step 3]

**Expected Behavior**:
[무엇이 일어나야 하는가]

**Actual Behavior**:
[실제로 무엇이 일어났는가]

**Environment**:
- Browser: Chrome 120
- OS: macOS 14
- Screen: 1920x1080

**Screenshots/Videos**:
[첨부]

**Priority**: Critical / High / Medium / Low
```

### 우선순위 기준

- **Critical**: 서비스 사용 불가, 결제 실패
- **High**: 주요 기능 오작동
- **Medium**: UI 깨짐, 부분 기능 오류
- **Low**: 텍스트 오타, 마이너 디자인 이슈
