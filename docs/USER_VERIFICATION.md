# 사용자 모드 검증 체크리스트 (E2E)

**목적:** 20명 동시 검증 시 사용자 관점 E2E 시나리오. 관리자 모드(/admin)와 별도로 수동 또는 자동화 전 체크용.

---

## 시나리오 1: 정상 플로우 (입력 → 결과 → 추천)

- [ ] 메인(/) 접속
- [ ] "오늘의 개소리" 배너 노출 (또는 로드 실패 시 미노출)
- [ ] 연·월·일·성별 입력 후 "내 실체 보기" 클릭
- [ ] 로딩 표시 후 결과 카드 노출 (동물명, 일주, 겉모습, 해시태그)
- [ ] "이 타입에게 추천 음식" 3종 노출
- [ ] "이 타입에게 추천 제품" 3종 노출

---

## 시나리오 2: 공유 → Lv.2 해금

- [ ] 결과 화면에서 "친구들이 보는 내 뒷모습" 섹션까지 스크롤
- [ ] "공유하고 뒷모습 보기" 클릭 → 공유 시트 또는 클립보드 복사
- [ ] "해금됨" 표시
- [ ] 결과 카드 클릭 시 뒷면(친구들이 보는 너) 노출

---

## 시나리오 3: Lv.3 블러 → 결제 CTA

- [ ] "새벽 2시의 본능" 섹션까지 스크롤
- [ ] 블러 처리된 영역 + "300원에 보기" 버튼 노출
- [ ] 버튼 클릭 → 결제(스텁) 모달 노출
- [ ] 닫기 또는 해금 후 블러 제거

---

## 시나리오 4: 예외 (유효성)

- [ ] 연·월·일 비움 → 제출 시 "연, 월, 일을 모두 입력하세요." 또는 비활성 버튼
- [ ] 2월 30일 입력 → 제출 시 "존재하지 않는 날짜예요."
- [ ] 미래 날짜 입력 → "생년월일은 오늘 이전이어야 해요."
- [ ] 숫자 아님 입력 → "숫자로 입력하세요."

---

## 시나리오 5: 페이지 접근

- [ ] `/payment/success` 접속 → 안내 문구 + 홈 링크
- [ ] `/payment/fail` 접속 → 안내 문구 + 홈 링크
- [ ] `/gift` 접속 → Phase 3 안내 + 홈 링크
- [ ] 존재하지 않는 경로 → 404 페이지 + 홈 링크

---

## 교차 검증 (20명)

- 각 시나리오를 서로 다른 환경(브라우저/기기)에서 1회 이상 수행
- 실패 항목은 [ERROR_CATALOG.md](./ERROR_CATALOG.md)와 매핑 후 이슈 등록
