# component-gen

Use for new React components or major component rewrites.

## Workflow
1. Read `docs/00-overview/CONTEXT_ENGINE.md` for placement rules.
2. Inspect nearby components in the same domain folder.
3. Implement the component in the correct domain path.
4. Keep primitives in `src/components/ui/`; keep business components in domain folders.
5. Run lint and targeted UI tests when behavior changes.
