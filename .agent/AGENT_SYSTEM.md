## Team Map
- T1 Planning: specs, roadmap, documentation governance.
- T2 Frontend: pages, components, UI behavior.
- T3 Backend: API routes, DB logic, payments.
- T4 Engine: saju and scoring logic.
- T5 Data: datasets, content, recommendation inputs.
- T6 Design: visual system and interaction polish.
- T7 QA: tests, regressions, release checks.
- T8 DevOps: deploy, env, infra automation.
- T9 Security: auth, OAuth, secrets, trust boundaries.
- T10 Growth: SEO, analytics, sharing, funnels.

## Dispatch Rules
- Use one primary team owner.
- Add a second team only when the task crosses a clear boundary.
- Prefer sequential work for backend, security, and deploy tasks.
- Prefer parallel work for frontend, design, and data tasks when files do not overlap.
